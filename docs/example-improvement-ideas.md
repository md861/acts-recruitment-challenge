# Example Improvement Ideas

These are examples only. Candidates do not need to choose one of these.

## Modelling

- Add destination-based movement.
- Add terrain costs and restricted cells.
- Add group behaviour.
- Add simple schedules.
- Add deterministic scenario loading from JSON.
- Add model metrics such as average distance to objective or congestion count.

## Integration / Fullstack

- Add API validation and clearer error responses.
- Add server-sent events for simulation snapshots.
- Add frontend controls for pause, step, reset, and speed.
- Add a scenario configuration editor.
- Add tests for model-to-API transformation.
- Add UI affordances for stale data or model failure.

## Platform

- Add Dockerfiles and docker-compose support.
- Add Kubernetes manifests or a Helm chart.
- Add readiness/liveness probes.
- Add ConfigMaps and Secrets where appropriate.
- Add a k3d runbook.
- Add structured logs or basic metrics.
