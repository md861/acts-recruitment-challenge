# Kubernetes Starter

This directory is intentionally light.

Platform candidates can use it as a starting point for a k3d or Kubernetes deployment. A good submission might add:

- Deployments for model, API, and frontend.
- Services for each component.
- ConfigMaps for simulation settings and API model URL.
- Liveness and readiness probes.
- A k3d runbook.
- A simple verification command sequence.

Keep it runnable and explain the trade-offs.
